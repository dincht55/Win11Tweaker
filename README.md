# Windows 11 設定精靈 (Win11Tweaker)

一套模組化、互動式的 Windows 11 優化設定工具。透過逐項提問的方式，協助使用者在**首次進入桌面後**，快速完成隱私強化、資源優化、個人化與系統清理，並將所有選擇保存為可重複使用的設定檔。

**完整支援繁體中文（台灣）與英文雙語介面**，啟動時可自由選擇。

\---

## 一、設計目標

本工具的核心目的是**設定 Windows，而非安裝第三方軟體**。聚焦於：

* **減少廢物**：停用不必要的服務、排程、預裝 App
* **強化隱私**：關閉遙測回傳、廣告追蹤、App 過度權限
* **節省資源**：關閉浪費效能的視覺特效與背景程序
* **個人化習慣**：副檔名顯示、右鍵選單、工作列、主題等
* **磁碟清理**：清除暫存、快取、更新殘留，釋放空間

涵蓋範圍以**最常見、最有價值的優化項**為主，聚焦於目前受支援的 Windows 11 版本（**24H2 / 25H2**，以 Build 26100 為基準線）。詳見「版本相容性」一節。

**收錄準則**：不追求「零副作用」，而是「**取捨可被使用者理解與掌控**」。每個涉及取捨的項目都會清楚說明代價，由使用者依自身用途決定。

## 二、核心特性

|特性|說明|
|-|-|
|**模組化多腳本**|14 大類各自獨立腳本，可單獨執行，互不影響|
|**完整雙語**|繁中（台灣）/ 英文，啟動時選擇；所有介面文字、項目名稱、說明、選項均雙語|
|**兩種執行模式**|完整 / 自訂，適應不同需求|
|**統一選項式介面**|每題以「問題 + 選項清單 + 建議」呈現，一律用數字鍵選擇|
|**Enter 跟隨建議**|直接按 Enter 即採用該題的建議選項，快速通過|
|**目前狀態偵測**|逐項顯示目前實際設定（登錄檔 / 排程 / 服務三型皆支援）|
|**動態掃描**|權限、服務、預裝軟體、磁碟清理等類即時掃描系統現況，只顯示實際存在的項目|
|**選項隨版本調整**|選項依系統 Build / Edition 動態過濾，只顯示對此系統有效的選項|

|**分類設定檔**|各類獨立匯出 JSON，可載入重複套用|
|**資料驅動**|設定項寫在資料檔（知識庫），與程式邏輯分離，易讀易改、方便擴充|

## 四、14 大類與執行順序

執行順序依「**低風險先做、可逆優先、不可逆與需重開機的殿後**」原則排定。

|執行序|類別|提問類型|風險|重開機|項數|
|:-:|-|:-:|:-:|:-:|:-:|
|01|隱私與遙測|逐題|低|-|12|
|02|App 權限管理|動態掃描|低|-|\~37|
|03|AI 功能|逐題|低|-|5|
|04|應用程式行為|逐題|低|-|11|
|05|個人化習慣|逐題|低|-|20|
|06|網路設定|逐題|中|⚠|4|
|07|硬體 / 裝置|逐題|中|⚠|3|
|08|視覺與效能|分組|中|⚠|17|
|09|安全性|逐題|中|⚠|3|
|10|排程工作|逐題|中|-|13|
|11|不必要服務|動態掃描|中高|⚠|\~59|
|12|系統行為|逐題|中高|⚠|5|
|13|預裝軟體移除|動態掃描|高|-|\~56|
|14|磁碟清理|動態掃描|高|-|8|

標「\~」者為知識庫收錄項數，實際顯示數量依系統掃描結果而定。

### 三種提問類型

* **逐題**（9 類）：每個設定項獨立提問，一題一個選項清單。用於彼此獨立、因人而異的設定。
* **分組**（1 類，僅 08 視覺與效能）：多個相關項共用一組「效能優先 / 外觀優先」整組套用，也可展開個別調整。用 `Show-Group` 引擎。
* **動態掃描**（4 類：02 / 11 / 13 / 14）：模組即時掃描系統現況（權限、服務、已安裝 App、可釋放空間），只列出實際存在的項目，再逐一詢問。知識庫作為「對照表」補上友善名稱、說明與建議。

### 排序原則說明

* **低風險先做**：純登錄檔、可逆的設定優先
* **不可逆殿後**：預裝軟體移除、磁碟清理放最後（避免誤刪後前功盡棄）
* **需重開機集中後段**：標 ⚠ 的類含需重開機才生效的設定
* **磁碟清理最後**：前面步驟會產生新暫存 / 快取，最後一次清乾淨

## 五、各類內容摘要

### 01 隱私與遙測（逐題）

系統層級的資料回傳與追蹤。含診斷資料等級、廣告識別碼、活動記錄、剪貼簿雲端同步、墨跡與輸入診斷、語音辨識、回饋頻率等。

### 02 App 權限管理（動態掃描）

掃描 `CapabilityAccessManager\ConsentStore`（HKLM + HKCU 合併），逐一顯示各權限（位置、相機、麥克風、通知存取、檔案系統等）的目前狀態並詢問開關。只管 Windows 商店（UWP）App，與第三方桌面程式無關。

### 03 AI 功能（逐題）

Recall 回顧、工作列 Copilot 按鈕、Click to Do、Copilot 實體鍵、設定 App AI 代理等 Windows 內建 AI 功能的控制。

### 04 應用程式行為（逐題）

OneDrive / Teams 開機自啟、Edge 各項行為（啟動增強、背景執行、首次體驗、新聞、購物助理、側邊欄）、自動播放、Xbox Game Bar 錄製等。

### 05 個人化習慣（逐題）

副檔名顯示、隱藏檔、檔案總管預設位置、右鍵選單樣式、主題、工作列對齊 / 搜尋框 / 各按鈕、桌面圖示、刪除確認等介面偏好。

### 06 網路設定（逐題）

LLMNR、NCSI 探測、WPAD 自動探索、NetBIOS over TCP/IP。多屬安全 / 隱私取捨，每項清楚說明代價（如停用後公共 Wi-Fi 登入頁可能跳不出），多數建議維持現狀。

### 07 硬體 / 裝置（逐題）

電源計畫、快速啟動（Fast Startup）、裝置中繼資料自動下載（OEM App / 圖示）。

### 08 視覺與效能（分組）

17 項視覺特效（動畫、陰影、平滑、透明等）。可整組一鍵「效能優先 / 外觀優先」，或展開個別調整每一項。

### 09 安全性（逐題）

記憶體完整性 HVCI（核心隔離）、UAC 等級、SmartScreen 信譽防護。均為因人而異的取捨，每項說明安全與便利 / 相容性的權衡。

### 10 排程工作（逐題）

Windows 背景排程工作，多屬遙測 / 資料收集類（CEIP、相容性評估、回饋上傳、地圖更新等）。可雙向停用 / 啟用。多數建議停用，少數（如磁碟診斷）建議維持。

### 11 不必要服務（動態掃描）

掃描系統服務，白名單機制：系統必要服務自動隱藏保護，只列出知識庫收錄的可調整服務。四選項（停用 / 手動 / 自動 / 維持），多數建議「手動」（需要時仍能觸發，比停用安全）。

### 12 系統行為（逐題）

GameDVR 背景錄製、視覺透明效果、剪貼簿歷程、背景 App 執行、系統廣告與建議內容（含各處推銷、廣告追蹤 ID）。

### 13 預裝軟體移除（動態掃描）

掃描已安裝 App（商店 UWP + 傳統 Win32）。系統關鍵元件黑名單自動隱藏。收錄項附中文名與移除建議；未收錄與第三方程式一律「建議保留」，由使用者自行判斷。

### 14 磁碟清理（動態掃描）

掃描並估算各項可釋放空間。分「安全清理區」（暫存、快取、更新殘留、回收筒，純賺）與「進階區」（Windows.old、WinSxS 元件清理，有明確取捨）。逐項詢問是否清理。

## 六、操作流程

### 6.1 啟動

以系統管理員身分開啟 PowerShell，執行：

```powershell
powershell -ExecutionPolicy Bypass -File .\Main.ps1
```

啟動後依序：**選擇語言 → 歡迎畫面 → 系統資訊偵測 → 命名本次設定檔 → 主選單**。

### 6.2 語言選擇

啟動第一步即詢問語言：

```
Language / 語言
  [1] 繁體中文 (台灣)
  [2] English
```

選定後全介面（含後續所有提問、說明、摘要）以該語言顯示。

### 6.3 兩種執行模式

主選單提供兩種模式：

```
  [1] 完整模式    依序執行 01 ~ 14 全部類別
  [2] 自訂模式    自選要執行的類別
  [Q] 離開
```

* **完整模式**：從 01 到 14 依序全跑。每題可直接按 Enter 採用建議，快速通過。
* **自訂模式**：顯示 14 類清單，輸入編號執行單一類別；執行完回到自訂選單可繼續挑其他類別，按 Q 返回主選單。

### 6.4 提問互動（統一選項式介面）

每個設定項以固定格式呈現：

```
────────────────────────────────────
類別名稱          進度：3 / 12
────────────────────────────────────
項目：<項目名稱>
說明：
  <這個設定是什麼、影響什麼>
目前：<目前實際設定值>
  [1] <選項一>（建議）
  [2] <選項二>
  [3] 維持現狀
備註：<延伸說明、注意事項、技術細節>
────────────────────────────────────
[1] [2] [3] [S]略過 [P]上一步 [Q]結束 (Enter=1) >
```

* **一律數字鍵**：所有選項用數字選擇，避免 Y/N 語意混淆（磁碟清理等「確認動作」例外，用 Y/N 較自然）。
* **Enter 跟隨建議**：提示列末端的 `(Enter=X)` 表示直接按 Enter 即採用第 X 個選項（該題建議值）。
* **目前狀態**：`目前：` 顯示掃描到的實際設定，若已是建議值，套用時會標示「已是此設定」。
* **S 略過 / P 上一步 / Q 結束**：隨時可跳過、回退或中斷（中斷會保存進度）。

### 6.5 分組提問（僅 08 視覺與效能）

分組類先顯示整組選項：

```
  視覺效果 (動畫、陰影、平滑等，共 17 項)  (群組 1/1)
  <群組說明>
  [1] 效能優先 (關閉視覺特效，系統較快)
  [2] 外觀優先 (開啟視覺特效，較美觀)
  [3] 展開個別調整
  [4] 維持現狀
      → 建議：效能優先 (直接 Enter 採用)
```

選 \[1]/\[2] 整組一鍵套用；選 \[3] 展開後逐一調整每個項目。

### 6.6 套用機制（以「類」為單位）

同一類的所有提問收集完後，才統一套用：

* 逐項寫入登錄檔 / 排程 / 服務，並讀回驗證
* 顯示**執行摘要**：已套用 / 已是原設定 / 失敗 / 略過 / 不支援
* 匯出該類設定檔（JSON）到 `Config\`

## 七、設定檔與續傳

### 7.1 設定檔命名

```
{電腦名}_{使用者}_{類別}_{時間戳}.json
例：RT3900X_Dincht_03_AI_20260706_073538.json
```

每類獨立匯出，記錄該類所有項目的選擇，可作為紀錄與稽核。涉及「刪除登錄檔值」的設定（如停止 OneDrive 開機自啟），會另在 `Restore\` 備份原值，日後可用 `Restore.ps1` 救回。

### 7.2 斷點續傳

執行中按 Q 中斷時，進度保存至 `Progress\Progress.json`。下次執行該類時會詢問：

```
偵測到上次未完成的進度。
  [Y] Resume  從上次中斷處接續
  [N] Restart 重新開始
```

## 八、輔助工具

除主程式外，專案根目錄提供兩支獨立工具，均可單獨執行、啟動時先選語言（中英）。

### 8.1 Restore.ps1（還原工具）

救回先前被「刪除類設定」移除的登錄檔值。當精靈執行了刪除類操作（如停止 OneDrive / Teams 開機自啟）時，會在刪除前把原值備份到 `Restore\`。此工具讀取這些備份，逐項寫回登錄檔。

* **安全原則**：若某個值「目前已存在」，自動略過不覆蓋（只補回缺少的值），避免蓋掉使用者後來自行設定的內容。
* **用法**：`powershell -ExecutionPolicy Bypass -File .\Restore.ps1`

### 8.2 Reset.ps1（重置工具）

清除精靈執行後產生的所有檔案，回到初始（剛下載）狀態：`Config\` 設定檔、`Progress\` 續傳記錄、`Reports\` 執行日誌、`Restore\` 還原備份。

* **先掃描列出**：執行前列出各目錄的檔案數與容量，不盲刪。
* **明確警告**：清除 `Restore\` 備份後，先前刪除類設定將**無法再還原**；此操作不可復原。
* **雙重確認**：需輸入大寫 `YES` 才實際執行，其他任意鍵取消。
* **保留結構**：`.gitkeep` 與說明用的 `README.md` 會保留，不破壞目錄結構。
* **用法**：`powershell -ExecutionPolicy Bypass -File .\Reset.ps1`

## 九、錯誤處理與驗證

* **寫入驗證**：每項套用後讀回確認，確保設定真正生效
* **版本相容偵測**：項目標 MinBuild / Edition，不符的系統自動略過（顯示「不支援」）
* **服務保護**：系統必要服務黑名單保護，受保護服務嘗試修改失敗時安全略過
* **檔案佔用略過**：磁碟清理遇使用中的檔案自動跳過，不影響執行中的程式
* **後台日誌**：所有動作即時寫入 `Reports\Execution_{時間戳}.log`（中文，供除錯追溯）

## 十、版本相容性

### 支援版本基準

以 **Windows 11 24H2（Build 26100）** 為文件基準，涵蓋 **24H2 與 25H2**。部分項目為特定版本引入（如 25H2 的設定 App AI 代理），會標註 MinBuild，在較舊系統上自動略過。

### 知識庫查證原則

* 每個設定項的登錄檔路徑、鍵值、政策名稱均經查證
* 涉及取捨的項目明確標示代價，不隱瞞副作用
* 收錄準則為「取捨可被使用者理解與掌控」，而非「零副作用」

## 十一、雙語架構（開發者參考）

本工具完整支援中英雙語，機制如下：

* **語言檔**：`Lang\lang_zh-TW.psd1` 與 `lang_en.psd1`，以鍵值對存放所有介面文字。`Get-Text "key"` 依當前語言取值，支援 `-Args @(...)` 格式化參數。
* **知識庫雙語欄位**：各 kb 項目同時有 `NameZh`/`NameEn`、`DescZh`/`DescEn`、`Note`/`NoteEn` 等欄位。引擎透過 `Get-Field -Obj $item -Base "Name"` 依當前語言選擇對應欄位，英文缺漏時自動 fallback 中文。
* **型別相容**：`Get-Field` 同時支援 `hashtable`（靜態 kb 項目）與 `pscustomobject`（動態掃描類即時建立的候選項）。
* **動態掃描類保留雙語**：02 / 11 / 13 等類在掃描建立候選項時，會一併保留英文欄位（如 `NameEn` / `NoteEn`），避免掃描後英文遺失。
* **日誌保持中文**：`Write-Log` 的後台日誌訊息固定中文，僅使用者可見文字翻譯，利於統一除錯。

\---

## 十二、擴充性（資料驅動設計）

新增設定項不需改動程式邏輯，只需編輯對應的 `Data\kb_XX.psd1`：

* 逐題類：在 kb 中新增一個項目（NameZh/NameEn、DescZh/DescEn、Choices、Recommend、Notes 等）
* 動態掃描類：在對照表新增一筆（Match / Key、雙語名稱、Recommend、Note）
* 每個 Choice 可掛多個登錄檔 / 排程 / 服務動作，引擎自動處理套用與驗證

## 十三、安全性原則

* **系統必要服務保護**：核心服務黑名單，絕不出現在可調整清單
* **系統關鍵 App 保護**：商店、執行庫、殼層等關鍵套件掃描時自動隱藏
* **不可逆操作明確警告**：Windows.old 清除、WinSxS 清理、預裝移除等，均清楚說明後果並要求確認
* **可還原**：所有設定型變更皆匯出設定檔留存；涉及刪除登錄檔值的操作另備份原值，可用 Restore.ps1 救回
